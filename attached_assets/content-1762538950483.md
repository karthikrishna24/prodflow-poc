# ReleaseStream - Release Management Dashboard

[Permalink: ReleaseStream - Release Management Dashboard](https://github.com/karthikrishna24/prodflow-poc#releasestream---release-management-dashboard)

A collaborative, editable flow dashboard for managing software releases across environments.

## Prerequisites

[Permalink: Prerequisites](https://github.com/karthikrishna24/prodflow-poc#prerequisites)

- Node.js (v20 or higher)
- PostgreSQL database (or Neon Database connection string)
- npm or yarn

## Setup Instructions

[Permalink: Setup Instructions](https://github.com/karthikrishna24/prodflow-poc#setup-instructions)

### 1\. Install Dependencies

[Permalink: 1. Install Dependencies](https://github.com/karthikrishna24/prodflow-poc#1-install-dependencies)

```
npm install
```

### 2\. Configure Database

[Permalink: 2. Configure Database](https://github.com/karthikrishna24/prodflow-poc#2-configure-database)

You need to set up a `DATABASE_URL` environment variable. The app uses Neon Database (PostgreSQL).

**Option A: Using a `.env` file (Recommended for local development)**

Create a `.env` file in the root directory:

```
DATABASE_URL=postgresql://user:password@host:port/database
```

**Option B: Set environment variable directly**

- **Windows PowerShell:**



```
$env:DATABASE_URL="postgresql://user:password@host:port/database"
```

- **Windows CMD:**



```
set DATABASE_URL=postgresql://user:password@host:port/database
```

- **Linux/Mac:**



```
export DATABASE_URL="postgresql://user:password@host:port/database"
```


### 3\. Initialize Database Schema

[Permalink: 3. Initialize Database Schema](https://github.com/karthikrishna24/prodflow-poc#3-initialize-database-schema)

Run the database migrations:

```
npm run db:push
```

### 4\. Start the Development Server

[Permalink: 4. Start the Development Server](https://github.com/karthikrishna24/prodflow-poc#4-start-the-development-server)

```
npm run dev
```

The application will start on **port 5000** (or the port specified in the `PORT` environment variable).

Open your browser and navigate to: `http://localhost:5000`

## Available Scripts

[Permalink: Available Scripts](https://github.com/karthikrishna24/prodflow-poc#available-scripts)

- `npm run dev` \- Start the development server
- `npm run build` \- Build the application for production
- `npm run start` \- Start the production server (requires build first)
- `npm run check` \- Run TypeScript type checking
- `npm run db:push` \- Push database schema changes to the database

## Project Structure

[Permalink: Project Structure](https://github.com/karthikrishna24/prodflow-poc#project-structure)

- `client/` \- React frontend application
- `server/` \- Express backend API
- `shared/` \- Shared TypeScript types and schemas
- `migrations/` \- Database migration files (generated by Drizzle)

## Environment Variables

[Permalink: Environment Variables](https://github.com/karthikrishna24/prodflow-poc#environment-variables)

- `DATABASE_URL` (required) - PostgreSQL connection string
- `PORT` (optional) - Server port (default: 5000)
- `NODE_ENV` \- Environment mode (development/production)

## Troubleshooting

[Permalink: Troubleshooting](https://github.com/karthikrishna24/prodflow-poc#troubleshooting)

### Database Connection Issues

[Permalink: Database Connection Issues](https://github.com/karthikrishna24/prodflow-poc#database-connection-issues)

If you see "DATABASE\_URL must be set", make sure you've set the environment variable correctly for your operating system.

### Port Already in Use

[Permalink: Port Already in Use](https://github.com/karthikrishna24/prodflow-poc#port-already-in-use)

If port 5000 is already in use, set the `PORT` environment variable to a different port:

```
# Windows PowerShell
$env:PORT="3000"
npm run dev

# Linux/Mac
PORT=3000 npm run dev
```
